# Java Study Notes 02: Basic Syntax

# 1. Comments in Java

## 1.1 Single-line Comments

```java
public class HelloWorld {
    public static void main(String[] args) {
        // Output a line of text: HelloWorld
        System.out.println("Hello, World");
    }
}
```

## 1.2 Multi-line Comments

```java
public class HelloWorld {
    public static void main(String[] args) {
        /*
            Output a line of text: HelloWorld
            Multi-line comment
         */
        System.out.println("Hello, World");
    }
}
```

## 1.3 Documentation Comments

```java
public class HelloWorld {
    public static void main(String[] args) {
        /**
         * @Description HelloWorld
         * @Author Aucki6144
         */
        System.out.println("Hello, World");
    }
}
```

**Writing comments is a good habit.**

## 2.2 Notes on Identifiers

- All identifiers should start with a letter (A-Z or a-z), dollar sign (`$`), or underscore (`_`).
- Characters after the first one can be any combination of letters, dollar signs, underscores, or digits.
- Keywords cannot be used as identifiers.
- Do not use Pinyin (use meaningful English words instead).
- Identifiers cannot be duplicated within the same scope.
- Identifiers are case-sensitive.

# 3. Java Data Types

**Java is a strongly typed language. Variable usage must strictly follow rules, and all variables must be defined before use.**

(Weakly typed languages: VB, JS, Python, etc.)

Java has two categories of data types:

- Primitive Types
- Reference Types

## 3.1 Primitive Types

### 3.1.1 Integer Types

- `byte`: -128 to 127 (1 byte)
- `short`: -32,768 to 32,767 (2 bytes)
- `int`: -2,147,483,648 to 2,147,483,647 (4 bytes)
- `long`: 8 bytes integer

### 3.1.2 Floating-Point Types

- `float`: 4 bytes floating-point
- `double`: 8 bytes floating-point

### 3.1.3 Character Type

- `char`: 2 bytes character

### 3.1.4 Other

- `boolean`: Boolean type (true/false)

**Example**

```java
int num1 = 10;
byte num2 = 20;
short num3 = 30;
long num4 = 40L; // long values should end with 'L'
float num5 = 50.1F; // float values should end with 'F'
double num6 = 3.141592653589;
char character = 'A';
boolean flag = true; // true or false
```

## 3.5.1 About Integers



Floating-point numbers are not precise; there are rounding errors.

**It is best to completely avoid using floating-point numbers for comparison.**

Alternative: Use the `BigDecimal` mathematical utility class.

# 4. Java Type Conversion

When performing operations with different data types, they must be converted to the same type first. Avoid overflow during conversion.

Low to High: Automatic (Implicit) Conversion
High to Low: Mandatory (Explicit) Conversion

Low -------------------------------------------> High
`byte`, `short`, `char` -> `int` -> `long` -> `float` -> `double`

**Points to Note**

- Boolean values cannot be converted.
- Object types cannot be converted to unrelated types.
- When converting from high capacity to low capacity, memory overflow or precision loss may occur.
- JDK 7 feature: Numbers can be separated by underscores for readability.

```java
int money = 10_0000_0000;
```

# 5. Variables in Java

**Variables are quantities that can change.**

Java is a strongly typed language; every variable must have a declared type.

A Java variable is the most basic storage unit in a program. Its elements include the variable name, variable type, and scope.

```java
type varName [=value] [{,varName[=value]}] ;
// It is recommended to declare one variable per line.
```

## 5.1 Variable Scope

```java
public class Variable {
    static int allClicks = 0; // Class variable (static)
    String str = "Hello , world"; // Instance variable

    public void method(){
        int i = 0; // Local variable

    }
}
```

- **Class Variables**: Belong to the class (static).

```java
public class Demo02 {
    static double salary = 250000;
    public static void main(String[] args) {
        System.out.println(salary);
    }
}
```

- **Instance Variables**: Belong to the object.

```java
public class Demo01 {
    String name = "aucki6144";
    int age;
    public static void main(String[] args) {
        Demo01 demo01 = new Demo01();
        System.out.println(demo01.name);
        System.out.println(demo01.age);
    }
}

/*
Output:
aucki6144
0
*/
```

*Except for primitive types, the default value for all other types is `null`.*
*For primitive types: numeric types default to 0 or 0.0, boolean defaults to false.*

- **Local Variables**: Defined inside methods. Must be declared and initialized before use. The scope is limited to the method where they are defined.

## 5.2 Constants

Values that cannot be changed.

```java
static final double PI = 3.14;
// or final static double ...
// Modifiers like final and static do not have a fixed order.
```

## 5.3 Naming Conventions

- **All variables, methods, class names**: Should be meaningful ("Self-documenting").
- **Class member variables**: First letter lowercase, camelCase.
- **Local variables**: First letter lowercase, camelCase.
- **Constants**: All uppercase, words separated by underscores (e.g., `MAX_VALUE`).
- **Class names**: First letter uppercase, PascalCase.
- **Method names**: First letter lowercase, camelCase.

# 6. Operators in Java

- **Arithmetic Operators**: `+`, `-`, `*`, `/`, `%`, `++`, `--`
- **Assignment Operator**: `=`
- **Relational Operators**: `>`, `<`, `>=`, `<=`, `==`, `!=`, `instanceof`
- **Logical Operators**: `&&`, `||`, `!`
- **Bitwise Operators**: `&`, `|`, `^`, `~`, `>>`, `<<`, `>>>`
- **Conditional Operator**: `? :`
- **Extended Assignment Operators**: `+=`, `-=`, `*=`, `/=`

*Bitwise operations are very efficient.*

# 7. Java Package Mechanism

A package is essentially a folder.

Generally, the domain name is inverted and used as the package name (e.g., `com.google.www`).

Importing a package:

```java
import java.util.Date;
```

# 8. Javadoc

The `javadoc` command is used to generate your own API documentation.

```java
/**
 * @author aucki6144
 * @version 1.0
 * @since 1.9
 */
public class Demo02 {
    static double salary = 250000;
    String name;

    /**
     * @param args Arguments
     * @throws Exception Exceptions
     */
    public static void main(String[] args) throws Exception {
        System.out.println(salary);
        // return name; // main is void, cannot return string.
    }
}
```

Command to generate Javadoc:

```bash
javadoc -encoding UTF-8 -charset UTF-8 Doc.java
```
